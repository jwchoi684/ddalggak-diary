# REQ-017: AI 채팅 — 활성 세션 (모드 C) + LLM 호출 플로우

## Source
docs/requirements.md §4.6.5 (모드 C), §4.6.7 (AI 호출 흐름), §4.6.8 (톤 유지), §4.6.9 (엣지케이스)

## Summary
페르소나 선택 직후 들어오는 활성 채팅 화면. **세션 컨텍스트는 완전 격리** — LLM 호출에 다른 대화 메시지는 전혀 포함하지 않고 `system prompt(페르소나) + 직렬화된 일기 전체 + 이번 세션 메시지만` 보낸다. 페르소나는 시작 시 고정. 응답에 인용 일기 칩 표시. 빈 대화는 저장 안 함.

## User Goal
선택한 페르소나의 톤으로 자기 일기에 대해 자연어로 질문하고 답을 받는다. 인용된 일기 칩을 탭해 즉시 해당 일기로 이동.

## Acceptance Criteria
- 헤더: 좌측 ‹ → 세션 종료(closed=true) + REQ-015로 복귀. 중앙: 페르소나 이모지+라벨(변경 진입점 없음). 우측: "완료" 버튼 = 명시적 종료.
- 첫 메시지 없을 때 예시 질문 칩 3~5개 표시. 칩 탭 → 입력창에 채움(자동 전송 아님).
- 사용자 메시지: 우측 정렬 primary 버블. AI 응답: 좌측 정렬 그레이 버블.
- LLM 호출 컨텍스트는 정확히 `[system: 공통 베이스 + persona.systemPrompt + 직렬화된 일기 전체, ...이 세션 메시지]`. 다른 세션 메시지는 포함 금지(grep으로 확인).
- 일기 직렬화 형식: `"[YYYY-MM-DD] 기분: 라벨(이모지) | 본문: ..."`.
- 모델: `gpt-4o-mini`(또는 출시 시점 최저가 mini). temperature 0.7, max_tokens 500.
- 응답에 날짜가 포함되면 일기 매칭 → `citedDiaryIds` 채우고 📌 칩 렌더. 칩 탭 → 해당 일기 에디터로 이동(REQ-009 + REQ-006으로 채팅 복귀).
- 세션 종료 시 isClosed=true로 저장. 단 **메시지 0개 세션은 저장 자체를 하지 않음**.
- API 키 보안:
  - Option A: 설정 화면에서 사용자 API 키 입력, 클라이언트 직접 호출 (본인용 v0 한정).
  - Option B/C: 서버리스 프록시 필수, 클라이언트에 키 노출 금지.
- 일기 0개 상태에서 진입 시: "아직 일기가 없어요. 먼저 일기를 써보세요" + 캘린더로 가는 버튼.
- API 호출 실패 시: 에러 버블 + "다시 시도" 버튼, 사용자 메시지 보존.
- 사용자가 "톤 바꿔줘" 같은 요청 시: systemPrompt의 가드(§4.6.8)가 모델이 거부하도록 유도. UI는 별도 검증 안 함 (MVP).

## Non-Goals
- 임베딩 기반 RAG (일기 양 폭증 시) — P2 (§8.3).
- 사용량 캡 UI — Option C 단계 별도.
- hallucination 후처리 — MVP 범위 외 (§4.6.9 #8).

## Dependencies
REQ-002, REQ-004, REQ-009, REQ-015, REQ-016

## Suggested Scope
- ActiveChatScreen + MessageBubble + CitedDiaryChip + 일기 직렬화 유틸 + LLM 호출 모듈(Option별 분기).
- 100줄 넘으면 분리.

## Expected Affected Areas
- Frontend: ActiveChatScreen + 메시지/칩 컴포넌트 + 호출 유틸
- Backend: Option B/C에서 프록시 함수 1개(`POST /api/chat`) — system+messages 받아 OpenAI 호출 + 응답 반환
- Database: 없음 (대화는 localStorage)
- Tests: 컨텍스트 격리(다른 세션 메시지 미포함), 직렬화 포맷, 빈 세션 비저장, 에러 시 메시지 보존
- E2E: 권장 — "새 대화→페르소나→질문→응답+칩→칩 탭→해당 일기"

## Risk Level
High — 격리 위반·키 노출·hallucination 어느 하나라도 사용자 신뢰를 한 번에 깨뜨림.

## Suggested Test Coverage
- LLM 호출 모듈의 messages 배열에 다른 세션 메시지가 포함되지 않는다 (mock으로 검증).
- 일기 직렬화 포맷이 명세와 정확히 일치.
- 메시지 0개 종료 시 storage에 추가되지 않음.
- 응답 텍스트의 날짜 추출 → 매칭되는 일기 id만 citedDiaryIds로.
