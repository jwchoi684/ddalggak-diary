# REQ-009: 일기 에디터 (단일 화면, 신규+편집 통합)

## Status
TODO

## Source
docs/requirements.md §4.3 (일기 에디터 전체, 단 §4.3.4 가로 캘린더와 §4.3.5 사진은 후속 REQ), §4.3.8 (진입 컨텍스트), §9 (엣지케이스)

## Summary
캘린더·리스트·AI 인용 어디서 진입해도 같은 에디터 컴포넌트가 뜬다. 헤더(뒤로가기 + 더보기 ⋯ 메뉴), 무드 이모지(탭→REQ-008 재오픈), 날짜 표시, 본문 textarea, 하단 툴바(갤러리는 REQ-011, 정렬 토글, 현재 시간 삽입, ✓ 저장), 1초 debounce 자동 저장 + 명시적 저장 시 토스트 "일기를 저장했어요!".

## User Goal
한 화면에서 신규/편집 구분 없이 일기를 쓰고 저장한다. 데이터 손실 위험을 자동 저장으로 제거한다.

## Acceptance Criteria
- 신규/편집 화면을 분리하지 않는다 (CLAUDE.md "Editor screen is one screen, not two").
- 진입 컨텍스트별 초기 상태가 §4.3.8 표와 일치:
  - 캘린더 빈 날짜 → 빈 필드 + 무드 모달 자동 + 삭제 메뉴 숨김.
  - 일기 있는 날짜/리스트/AI 인용 → 기존 데이터 채워짐 + 모달 호출 안 함 + 삭제 메뉴 표시.
- 헤더: 좌측 뒤로가기 → REQ-006 history 복귀. 우측 ⋯ → 시트로 "📋 일기 리스트", "🗑 삭제" (저장된 일기만).
- 무드 이모지(MoodIcon size=64+) 탭 → REQ-008 mode="change"로 오픈.
- 텍스트 정렬 토글(왼쪽/가운데) 1개 아이콘.
- "🕐 현재 시간" 탭 → 커서 위치에 "HH:MM " 삽입.
- ✓ 저장 아이콘은 에디터 focus + 변경사항 있을 때만 표시. 탭 시 저장 + 토스트.
- 자동 저장은 1초 debounce, 토스트는 띄우지 않음.
- 미저장 변경 상태로 뒤로 → 확인 모달 후 저장/취소.
- 같은 날짜에 이미 일기 있으면 신규 진입이 아니라 기존 데이터로 채워진 에디터 (PRD §9, 1일 1일기).
- 미래 날짜 작성 허용.

## Non-Goals
- 가로 캘린더 인라인 드롭다운 — REQ-010.
- 사진 카로젤 + 길게 누름 삭제 — REQ-011.
- 사진 전체화면 뷰어 — REQ-012.
- 텍스트 검색·필터 — v2.

## Dependencies
REQ-002, REQ-003, REQ-005, REQ-006, REQ-008

## Suggested Scope
- 에디터 컨테이너 컴포넌트 + 헤더 + 본문 + 툴바를 책임 단위로 분리. 합쳐서 100줄 넘으면 즉시 쪼개기 (CLAUDE.md File size).

## Expected Affected Areas
- Frontend: Editor 컨테이너 + 하위 컴포넌트들
- Backend: 없음
- Database: 없음
- Tests: 진입 컨텍스트별 4가지 초기 상태, 자동 저장 debounce, 저장 토스트, 미저장 변경 가드
- E2E: 핵심 — "캘린더 빈 셀→무드 선택→본문 입력→자동 저장→뒤로→캘린더에 무드 표시"

## Risk Level
High — 앱의 핵심 입력 경로. 데이터 손실 시 사용자 신뢰 즉시 붕괴.

## Suggested Test Coverage
- 4가지 진입 컨텍스트별 초기 상태.
- 본문 입력 1초 debounce 후 localStorage(REQ-002)에 upsert.
- ✓ 저장 시 토스트 노출, 자동 저장 시 토스트 없음.
- 미저장 변경 + 뒤로 → 확인 모달.

PRD §13.2의 "에디터 우측 헤더" 결정 사항: ⋯ 단일 메뉴 권장(레퍼런스 일치). 다른 패턴 채택 시 사용자 재확인 필요.
