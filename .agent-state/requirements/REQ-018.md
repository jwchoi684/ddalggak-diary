# REQ-018: AI 채팅 — 과거 대화 읽기 (모드 D)

## Status
DONE

## Source
docs/requirements.md §4.6.6 (모드 D 읽기 전용)

## Summary
대화 리스트 카드 탭 시 진입하는 읽기 전용 뷰. 모드 C와 거의 같지만 **입력창 자체가 없음**(숨김이 아니라 제거). 헤더 중앙에 "(종료됨)" 표시, 인용 일기 칩 탭은 정상 동작, 옵션으로 우측 상단 🗑 삭제.

## User Goal
지난 대화를 처음부터 다시 읽고, 거기서 언급된 일기로 점프한다. 새 질문이 필요하면 "새 대화" 버튼으로 자연스럽게 유도.

## Acceptance Criteria
- 입력창 컴포넌트 자체가 트리에 없음(`display:none`이 아님).
- 헤더 중앙: "👯 친구 (종료됨)" 형태.
- 헤더 우측: (옵션) 🗑 → 확인 모달 후 대화 영구 삭제 → REQ-015로 복귀.
- 초기 스크롤 위치: 최상단(대화 회상용).
- 인용 일기 칩 탭 → REQ-009 에디터 이동(REQ-006으로 채팅 복귀).
- 하단 안내 배너(옵션): "종료된 대화입니다. 추가 질문은 새 대화에서 해주세요." + 새 대화 버튼 → REQ-016.

## Non-Goals
- "이 대화 이어가기" — 명시적으로 막혀 있음. UI에 진입점 없음.
- 메시지 단건 삭제·편집 — v2.

## Dependencies
REQ-015, REQ-017

## Suggested Scope
- ReadOnlyChatScreen — REQ-017의 메시지 영역만 재사용, 입력창은 mount 안 함.

## Expected Affected Areas
- Frontend: ReadOnlyChatScreen (또는 ActiveChatScreen의 readOnly prop)
- Backend: 없음
- Database: 없음
- Tests: 입력창 없음 검증, 칩 탭 시 라우팅, 삭제 시 storage에서 제거
- E2E: 선택 — "리스트→과거 대화→칩→일기→뒤로→과거 대화"

## Risk Level
Low.

## Suggested Test Coverage
- 입력 가능한 요소(`textarea`, `input[type=text]`)가 DOM에 없다.
- 삭제 확인 후 conversations storage에서 해당 id 제거.
