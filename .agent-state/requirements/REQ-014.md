# REQ-014: 통계 화면

## Status
DONE

## Source
docs/requirements.md §4.7 (통계 화면 전체)

## Summary
헤더 우측 상단 ✕ 닫기, 중앙 상단에 연도(작음) + "M월" 좌우 화살표. 월 표시 아래 그 달에 등장한 무드 아이콘만 가로로 노출. 본문은 무드별 횟수를 가로 막대 그래프(둥근 끝, 무드 컬러)로 횟수 내림차순 표시.

## User Goal
"이번 달 내 감정 분포"를 한 화면에서 본다. 어떤 감정이 우세했는지 시각적으로 파악.

## Acceptance Criteria
- ✕ 탭 → REQ-006 이전 화면 복귀.
- 월 네비(‹ M월 ›) 디폴트는 진입 시 보고 있던 달.
- 등장 무드 행: 해당 달에 한 번 이상 등장한 MoodIcon만, 횟수 내림차순 또는 등장 순.
- 가로 막대 그래프: 각 행에 MoodIcon(40px) + 컬러 막대(끝 둥글게, radius=height/2) + 우측 횟수.
- 막대 컬러는 REQ-003의 `color` 사용.
- 횟수 내림차순.
- 빈 달: "이 달에는 기록이 없어요" + 등장 무드/그래프 숨김. 월 네비는 유지.

## Non-Goals
- 추가 인사이트(주별 추이, 작년 비교, streak) — v2 (§4.7.6).
- 다운로드/공유 — v2.

## Dependencies
REQ-002, REQ-003, REQ-005, REQ-006

## Suggested Scope
- StatsScreen + HorizontalBarChart 컴포넌트(차트 라이브러리 또는 단순 div+width%).
- Option A: Chart.js CDN, Option B: Recharts, Option C: Recharts 또는 서버 사이드 prerender 옵션.

## Expected Affected Areas
- Frontend: StatsScreen + 차트 컴포넌트
- Backend: 없음
- Database: 없음
- Tests: 집계 정확성, 빈 달 처리, 내림차순 정렬
- E2E: 선택 — 캘린더→📊→통계 노출

## Risk Level
Low.

## Suggested Test Coverage
- 일기 5개(기쁨 3, 슬픔 2)인 달에서 막대 길이·횟수가 맞음.
- 0개인 달은 빈 상태 노출 + 월 네비 살아있음.
- 등장 무드 행에 한 번도 안 쓴 무드는 안 나옴.

## Notes
컬러는 REQ-003의 무드 컬러 토큰 그대로. 별도 컬러 매핑 만들지 말 것.
