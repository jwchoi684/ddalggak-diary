# REQ-010: 에디터 내 가로 캘린더 인라인 드롭다운

## Status
TODO

## Source
docs/requirements.md §4.3.4 (날짜 영역 + 가로 캘린더)

## Summary
에디터 날짜 텍스트 탭 시 그 자리에 인라인으로 펼쳐지는 좌우 스크롤 날짜 띠. 한 번에 7~10일이 보이고, 일기 있는 날은 MoodIcon, 없는 날은 숫자. 다른 날짜 탭 시 현재 일기를 자동 저장하고 그 날짜로 에디터 컨텍스트를 전환.

## User Goal
에디터에 머문 채로 인근 날짜 일기를 빠르게 오갈 수 있다. 캘린더로 돌아갔다 다시 들어오는 마찰을 없앤다.

## Acceptance Criteria
- 날짜 텍스트 우측 `▾` 화살표 탭 시 드롭다운 펼침/접힘.
- 좌우 스크롤로 날짜 탐색.
- 일기 있는 날: MoodIcon(REQ-003) 작은 사이즈. 없는 날: 숫자.
- 선택된 날짜는 강조.
- 다른 날짜 탭 시: 현재 변경사항 자동 저장(REQ-009의 저장 로직 재사용) → 그 날짜의 에디터 상태로 전환 (없으면 빈 상태 + 무드 모달).
- 자동 저장 없이 변경 손실되는 경로 없음.

## Non-Goals
- 월/연 picker — P1.
- 가상 스크롤 — 불필요.

## Dependencies
REQ-002, REQ-003, REQ-005, REQ-009

## Suggested Scope
- 인라인 드롭다운 컴포넌트 1개 + 에디터에서 토글 상태 관리.

## Expected Affected Areas
- Frontend: HorizontalDatePicker 컴포넌트, 에디터 통합
- Backend: 없음
- Database: 없음
- Tests: 일기 유무에 따른 셀 표시, 다른 날짜 선택 시 저장→전환 순서
- E2E: 권장 — 에디터에서 다른 날짜 이동 후 첫 날짜 일기가 보존되는지

## Risk Level
Medium — 저장과 컨텍스트 전환 순서가 어긋나면 데이터 손실 위험.

## Suggested Test Coverage
- 변경사항이 있는 채로 다른 날짜 탭 → 첫 날짜 저장 호출 후 두 번째 날짜 로드.
- 일기 있는 날짜·없는 날짜 셀이 올바르게 그려짐.
