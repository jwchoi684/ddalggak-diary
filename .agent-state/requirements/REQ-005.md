# REQ-005: 디자인 시스템 프리미티브

## Status

## Source
docs/requirements.md §1.6 (디자인 언어 전체), §5 (공통 컴포넌트), §1.6.6 (컴포넌트 스타일 테이블)

## Summary
PRD 전반에서 반복되는 시각 패턴 — 흰 원형 헤더 버튼, 검정 FAB, 흰 카드, 바텀시트, 알약형 토스트, 확인 다이얼로그, Empty State — 를 디자인 시스템 폴더(또는 Option A 섹션)에 한 번씩 만들어 모든 화면이 재사용한다.

## User Goal
이후 REQ에서 새 화면을 만들 때 "흰 원형 아이콘 버튼 2개 헤더" 같은 패턴을 즉시 조립할 수 있고, radius·shadow·터치 타깃 44×44가 자동으로 지켜진다.

## Acceptance Criteria
- 다음 컴포넌트가 모두 존재: `IconButton`(흰 원형 44px), `Card`(흰 배경 radius 16~20 shadow y=2 blur=8 op=0.04), `FAB`(검정 56px), `BottomSheet`(상단 라운드 24px + 그립 핸들), `Toast`(알약형 1.5~2초), `ConfirmDialog`, `EmptyState`.
- 모든 인터랙티브 컴포넌트의 최소 터치 타깃이 44×44 (§1.6.7).
- 컬러는 REQ-001의 토큰 참조 (하드코딩 금지).
- Storybook 또는 단순 데모 페이지에서 모두 렌더 확인 (선택).

## Non-Goals
- 화면 단위 컴포넌트 (캘린더·에디터 등).
- 다크모드 — P1 (§8.2).

## Dependencies
REQ-001

## Suggested Scope
- Option B/C: `src/design-system/` 폴더에 컴포넌트당 파일 1개.
- Option A: HTML 안 `// ===== Components =====` 섹션에 함수/클래스 그룹.

## Expected Affected Areas
- Frontend: 디자인 시스템 폴더/섹션
- Backend: 없음
- Database: 없음
- Tests: 각 컴포넌트 렌더 + 핵심 prop 동작
- E2E: 없음

## Risk Level
Medium — 여기서 정한 prop 시그니처가 모든 화면에 굳음. 너무 좁게 만들면 REQ-007부터 우회 코드가 생김.

## Suggested Test Coverage
- 모든 컴포넌트가 정의된 토큰으로만 색·radius·shadow를 사용한다 (스타일 인스펙션).
- IconButton·FAB 클릭 시 onClick prop 호출.
- Toast가 지정된 ms 후 사라진다.

CLAUDE.md "UI 컴포넌트 재사용 규칙" 항목 1·2의 출발점. 이 REQ 이후 모든 신규 UI는 먼저 이 폴더부터 확인해야 함.
