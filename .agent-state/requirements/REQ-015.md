## Source
docs/requirements.md §4.6.1~§4.6.3 (컨셉 + 모드 A 대화 리스트)

## Summary
헤더 🔍 진입 시 첫 화면. 최상단에 "➕ 새 대화 시작" 버튼 고정, 그 아래로 과거 대화 카드들이 `lastMessageAt` 내림차순. 각 카드: 페르소나 이모지·라벨 + 상대 시간 + 첫 사용자 질문 30자.

## User Goal
지난 AI 대화들을 회상하거나 새 대화를 시작하는 출입구.

## Acceptance Criteria
- 헤더 좌측 ‹ → REQ-006 이전 화면 복귀.
- "새 대화 시작" 버튼 탭 → REQ-016(페르소나 선택 모달) 오픈.
- 대화 카드 탭 → REQ-018(과거 대화 읽기) 진입.
- 카드 상단: 페르소나 이모지+라벨 + " · " + 상대 시간("방금"/"2시간 전"/"어제"/"3일 전"/"YYYY.M.D").
- 카드 본문: 첫 사용자 메시지 앞 30자 정도.
- 빈 상태: "아직 대화가 없어요. AI에게 일기에 대해 물어보세요" + 새 대화 버튼만 표시.

## Non-Goals
- 대화 검색, 필터링 — v2.
- 대화 리스트 자체 삭제 — REQ-018에서 카드별 삭제는 옵션.

## Dependencies
REQ-002, REQ-004, REQ-005, REQ-006

## Suggested Scope

## Expected Affected Areas

- Backend: 없음

- E2E: 선택 — 캘린더→🔍→리스트→새 대화 버튼

Low.

- 빈 메시지 대화는 리스트에 없음 (REQ-017이 저장 자체를 안 함).
