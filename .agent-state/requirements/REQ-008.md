# REQ-008: 무드 선택 바텀시트 모달

## Status
DONE

## Source
docs/requirements.md §4.2 (기분 선택 모달 전체), §4.2.1 (트리거), §4.2.3 (인터랙션)

## Summary
바텀시트 형태(상단 라운드 + 그립 핸들)로 화면 60~80% 높이까지 올라오는 모달. 무드 10종을 3열 그리드로 배치(MoodIcon 72px + 라벨). 두 진입 시점(신규 작성 자동 호출 / 에디터에서 무드 탭) 모두 같은 UI를 쓰되, 닫기 시 동작만 다르다.

## User Goal
"오늘 기분이 뭐였지?"를 1탭으로 선택해 일기 작성을 시작한다. 마찰 최소화의 핵심.

## Acceptance Criteria
- BottomSheet 컴포넌트(REQ-005) 위에 조립.
- 상단: 그립 핸들 + "YYYY.MM.DD 요일" + "오늘은 어떤 하루였나요?".
- 상위 탭 "기본/테마" 중 "기본"만 활성, 서브탭 "기분/일상" 중 "기분"만 활성. 비활성 탭 탭 시 "곧 만나요!" 토스트.
- 무드 10종(REQ-003) 3열 그리드, 셀 간 16~24px.
- 항목 탭 → 무드 저장 + 모달 슬라이드 다운 + 에디터 복귀.
- 닫기(그립 드래그 / 뒷배경 탭 / X 버튼) 동작 분기:
  - **신규 작성 자동 호출** 경로: 모달 닫힘 + 에디터에서 한 단계 더 뒤로 (기분 미선택 상태로 머무르지 않음).
  - **기분 변경** 경로: 모달만 닫힘, 기존 기분 유지.

## Non-Goals
- 테마/일상 카테고리 — v2 (§3.4.1).
- 사용자 정의 무드 — 범위 외.

## Dependencies
REQ-002, REQ-003, REQ-005

## Suggested Scope
- 모달 컴포넌트 1개 + props로 두 진입 모드 구분.

## Expected Affected Areas
- Frontend: MoodPickerSheet 컴포넌트
- Backend: 없음
- Database: 없음
- Tests: 두 진입 모드별 닫기 동작 차이, 탭 활성/비활성, 항목 탭 시 콜백
- E2E: 캘린더 빈 셀→에디터→모달 자동 → 무드 선택 → 본문 작성으로 이어지는 흐름의 일부

## Risk Level
Low — UI·인터랙션 분기만 명확하면 됨.

## Suggested Test Coverage
- mode="initial" 닫기 시 onCancelInitial 콜백 호출.
- mode="change" 닫기 시 기분 변경 없음.
- 비활성 탭 탭 시 토스트 호출.

## Notes
PRD §4.2.1의 "두 경우 모두 같은 모달" 원칙 유지 — 절대로 두 컴포넌트로 분리하지 말 것.
