# REQ-011: 사진 추가 / 카로젤 / 길게 누름 삭제

## Status
DONE

## Source
docs/requirements.md §4.3.5 (사진 카로젤), §3.2 (Photo 타입), §9 (사진 최대 10장)

## Summary
에디터 하단 툴바의 🖼 갤러리 아이콘으로 사진 추가, 가로 스크롤 카로젤(정사각 80~96px 썸네일, 8px 간격)에 노출, 길게 누름(500ms) 시 삭제 오버레이 + 햅틱. 최대 10장 제한. MVP는 base64로 일기 객체 안에 직접 저장.

## User Goal
일기에 사진을 첨부해 그날의 시각적 기억을 남긴다. 마찰 없이 추가하고 실수로 삭제되지 않게 한다.

## Acceptance Criteria
- 🖼 탭 → 파일 선택 다이얼로그 (`accept="image/*"`).
- 선택된 이미지가 base64로 변환되어 일기의 `photos` 배열에 추가 (REQ-002 스키마).
- 카로젤은 가로 스크롤, 정사각 80~96px, 간격 8px.
- 사진 없음 상태에서는 카로젤 영역 자체 숨김(높이 0).
- 길게 누름 500ms → 해당 썸네일 위에 "삭제" 오버레이 + 모바일 햅틱 진동.
- 삭제 탭 → 즉시 삭제(또는 1회 확인), 오버레이 사라짐.
- 다른 곳 탭 → 오버레이 사라짐.
- 사진 수 10장 도달 시 🖼 비활성화 또는 "최대 10장" 안내.
- 사진 변경도 REQ-009의 자동 저장 경로를 탐.

## Non-Goals
- 사진 전체화면 뷰어 — REQ-012.
- 썸네일 별도 저장(메모리 최적화) — v2 (§7.3).
- 카메라 직접 촬영 — 파일 선택만으로 충분.
- IndexedDB로 마이그레이션 — v2.

## Dependencies
REQ-002, REQ-009

## Suggested Scope
- PhotoCarousel 컴포넌트 + base64 변환 유틸 + 길게 누름 훅.

## Expected Affected Areas
- Frontend: PhotoCarousel + 갤러리 입력 + 길게 누름 훅
- Backend: 없음
- Database: 없음
- Tests: 파일 선택 → base64 변환 → 배열 push, 길게 누름 500ms 후 오버레이, 10장 제한
- E2E: 권장 — 사진 추가→저장→재진입 시 사진 보존

## Risk Level
Medium-High — base64 + localStorage 5MB 제한과 직결. 큰 사진 여러 장이면 한계 초과 가능.

## Suggested Test Coverage
- 11번째 추가 시도 시 거부.
- 길게 누름 500ms 미만 → 오버레이 안 나타남.
- 사진 삭제 시 배열에서 제거 + 카로젤 재렌더.

## Notes
PRD §3.9 — localStorage 5MB 제한 주의. 큰 이미지 업로드 시 사용자에게 경고하거나 자동 리사이즈 검토(범위 외지만 위험 명시).
