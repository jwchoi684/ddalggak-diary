## Summary
PRD §3에 정의된 타입(`DiaryEntry`, `Photo`, `Mood`, `Persona`, `SearchConversation`, `ChatMessage`)을 코드로 옮기고, `ddalkkak:diaries:v1` / `ddalkkak:conversations:v1` / `ddalkkak:settings:v1` 세 키에 대해 read/write/upsert/delete를 제공하는 단일 저장 모듈을 만든다.

## User Goal
모든 화면이 같은 데이터 형식으로 읽고 쓰며, 키 이름·직렬화 방식이 한 곳에서 관리되어 v2에서 IndexedDB로 옮길 때 호출부 변경이 없게 만든다.

## Acceptance Criteria
- 타입 정의가 PRD §3.1~§3.7과 완전히 일치 (필드명·옵셔널 여부 포함).
- localStorage 키는 정확히 `ddalkkak:diaries:v1`, `ddalkkak:conversations:v1`, `ddalkkak:settings:v1`.
- 같은 `date`로 두 번 upsert하면 1개만 남는다 (PRD §9 — 1일 1일기).
- 손상된 JSON이 들어있어도 throw하지 않고 빈 배열로 폴백한다.
- 저장 모듈을 import한 호출부가 직접 `localStorage`에 접근하지 않는다 (grep으로 확인).

## Non-Goals
- 실제 화면 렌더링.
- 사진 base64 압축·썸네일 생성 (REQ-011 시 검토).
- 마이그레이션 로직 (v1만 존재).
