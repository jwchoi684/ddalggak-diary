# REQ-013: 일기 리스트 화면

## Status
DONE

## Source
docs/requirements.md §4.5 (일기 리스트 화면 전체), §4.5.3 (월 필터링), §4.5.4 (리스트 카드)

## Summary
헤더에 ‹ 뒤로가기 / `‹ YYYY년 M월 ›` 월 네비 / 정렬 토글(↓ 최신순 / ↑ 오래된 순). 본문은 선택된 달의 일기만 카드로 나열. 카드 구성: 상단 중앙 MoodIcon(64px) → 날짜 → 본문(최대 3줄, 초과 시 `...`) → 사진 썸네일 가로 나열(최대 3개 + `+N`).

## User Goal
달력 형식이 아닌 시간순 리스트로 일기를 훑어보고, 카드 탭으로 해당 일기 에디터에 진입.

## Acceptance Criteria
- 헤더 좌측: ‹ → REQ-006 이전 화면 복귀.
- 헤더 중앙: 월 네비 (`‹` 이전 / "YYYY년 M월" / `›` 다음). 디폴트는 캘린더가 보고 있던 달 (REQ-006에서 전달).
- 헤더 우측: 정렬 토글 라벨 + 화살표. 탭 시 두 상태 토글, 즉시 재정렬.
- 본문은 그 달 일기만 렌더, `date` 기준 정렬.
- 카드 구조: MoodIcon top-center → 날짜(회색, 작음) → 본문 3줄까지 → 사진 3개 + `+N`.
- 빈 본문: "(내용 없음)" 또는 사진/기분만 표시.
- 빈 달: "이 달에는 작성된 일기가 없어요" + 캘린더로 가는 버튼. 월 네비는 유지.
- 카드 탭 → 에디터(REQ-009) 진입 + 뒤로가기 시 리스트로 복귀(REQ-006).
- 정렬 상태는 세션 내 유지. 새로고침 시 "최신순"으로 리셋.

## Non-Goals
- 무드 필터, 텍스트 검색 — v2 (§4.5.7).
- 정렬 영속화 (설정) — v2.
- 다달 무한 스크롤 — v2.

## Dependencies
REQ-002, REQ-003, REQ-005, REQ-006

## Suggested Scope
- 리스트 화면 + 카드 컴포넌트(디자인 시스템 Card 재사용). 100줄 넘으면 분리.

## Expected Affected Areas
- Frontend: ListScreen, DiaryListCard
- Backend: 없음
- Database: 없음
- Tests: 월 필터링, 정렬 토글, +N 라벨, 빈 상태
- E2E: 권장 — "캘린더→리스트→카드 탭→에디터→뒤로 리스트(상태 보존)"

## Risk Level
Low-Medium.

## Suggested Test Coverage
- 2개월에 걸친 일기 데이터에서 한 달만 노출됨.
- 정렬 토글 시 순서 반전.
- 사진 4장 이상 시 `+N` 표기.
- 빈 본문 일기가 안전하게 렌더.

## Notes
