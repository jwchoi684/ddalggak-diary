## Source
docs/requirements.md §4.6.4 (모드 B 페르소나 선택 모달)

## Summary
대화 리스트의 "새 대화 시작" 버튼 탭 시 열리는 풀스크린 또는 바텀시트 모달. 타이틀 "어떤 톤으로 대화할까요?" + 14개 페르소나를 2열 그리드로 노출. 항목 탭 시 모달 닫힘 + REQ-017(신규 채팅) 진입.

## User Goal
대화 톤을 1탭으로 정해 시작한다. 시작 후에는 톤 변경이 불가하다는 모델을 사용자가 자연스럽게 학습.

## Acceptance Criteria
- 트리거: REQ-015의 "새 대화 시작".
- 레이아웃: 2열 × 7행 그리드, 항목당 (큰 이모지 + 라벨 + 한 줄 설명).
- 페르소나 데이터는 REQ-004에서 14종 모두.
- 항목 탭 → 모달 닫힘 + REQ-017로 personaId 전달.
- ✕ 탭 → 모달 닫힘 + REQ-015로 복귀.

## Non-Goals
- 페르소나 즐겨찾기·정렬 — v2.
- 사용자 정의 페르소나 — 범위 외.

## Dependencies
REQ-004, REQ-005, REQ-015

## Suggested Scope
- PersonaPickerModal 컴포넌트 1개.

## Expected Affected Areas
- Frontend: PersonaPickerModal
- Backend: 없음
- Database: 없음
- Tests: 14개 항목 모두 노출, 항목 탭 시 personaId 콜백, 닫기 동작
- E2E: 함께 — REQ-017과 묶어서 "새 대화→페르소나→채팅"

## Risk Level
Low.

## Suggested Test Coverage
- 14개 페르소나 모두 렌더.
- 항목 탭 시 onSelect(personaId) 호출.
- 모달 닫기는 REQ-015로만 복귀 (다른 화면으로 가지 않음).

## Notes
페르소나가 선택된 직후 REQ-017 화면 헤더에서 락 표시 — 모달 자체에 "한 번 정하면 변경 불가" 안내 문구는 선택 사항.
