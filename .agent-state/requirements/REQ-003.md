docs/requirements.md §3.4 (기분 마스터 리스트 10종), §1.6.4 (무드 아이콘 가이드)

## Summary
PRD §3.4의 10개 무드(joy/love/excited/calm/grateful/sad/angry/anxious/tired/embarrassed)를 id·emoji·label·color로 정의하고, 사이즈 prop만 받는 단일 `MoodIcon` 컴포넌트를 만든다. MVP는 emoji placeholder이며, 후에 수채화 일러스트로 **컴포넌트 내부만 교체**하면 끝나도록 경계를 한 곳에 둔다.

## User Goal
캘린더 셀·에디터 헤더·리스트 카드·통계 그래프 어디서든 같은 컴포넌트 한 줄로 무드를 그리고, 일러스트 교체 시 호출부를 건드리지 않는다.

## Acceptance Criteria
- 무드 10종 데이터가 정확히 PRD §3.4 표와 일치 (id·emoji·label·color HEX).
- 컬러는 전부 파스텔 채도 (CLAUDE.md "Confirmed decisions").
- `MoodIcon` 컴포넌트는 `id` + `size`(예: 32/48/64/72/80) prop만 받는다. 컬러·이모지 등 내부 결정 사항은 prop으로 노출하지 않는다.
- 호출부에서 직접 emoji 문자열을 쓰지 않는다 (grep `'😊'`/`'😢'` 등 안 나옴).
- `aria-label`이 label(한국어)로 자동 설정됨 (§7.2 접근성).

## Non-Goals
- 카테고리 확장 (테마/일상) — v2 (§3.4.1).
- 수채화 일러스트 자산 — 정식 출시 전 별도.
