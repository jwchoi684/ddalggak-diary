## Summary
14개 페르소나(friend, lover, sibling, junior, senior, employee, boss, king, mother, father, grandma, therapist, daoist, shaman)를 id·emoji·label·shortDesc·systemPrompt·sampleGreeting으로 정의한다. system prompt는 §3.8.1의 공통 베이스 + 페르소나별 톤 + 안전장치(사용자 모욕·비하 금지) 구조를 따른다.

## User Goal
AI 채팅 화면(REQ-016/017)이 페르소나 id만 받으면 톤·system prompt·예시 인사·아이콘을 한 번에 얻을 수 있다.

## Acceptance Criteria
- 페르소나 14종 데이터가 PRD §3.8 표와 정확히 일치 (id·emoji·label·한 줄 설명).
- 각 systemPrompt는 §3.8.1의 공통 베이스 + 페르소나별 톤 + "사용자를 존중하는 선을 항상 지킬 것" 안전장치를 포함.
- 페르소나가 시작 시 잠긴다는 가드("이 대화의 톤은 시작 시 정해져 있어요...")가 systemPrompt에 포함 (§4.6.8).
- shaman 페르소나에 미래 단정·불안 조성 금지 가드 포함 (§3.8.1).
- 호칭 기본값(sibling=언니, mother=우리 아이, etc.)이 systemPrompt에 명시.
