## Summary
캘린더를 최상위로 두고, 모든 화면 간 이동을 브라우저 history-stack 기반으로 처리한다. 같은 에디터 화면이 캘린더·리스트·AI 채팅 인용 어디서 진입했는지에 따라 뒤로가기 목적지가 달라지며, 복귀 시 이전 화면의 스크롤 위치·선택된 달·정렬 상태가 보존된다.

## User Goal
사용자가 "내가 어디서 왔는지" 자연스럽게 돌아갈 수 있고, 다른 화면으로 우회해도 직전 컨텍스트가 사라지지 않는다.

## Acceptance Criteria
- 라우트 5개: `/`(캘린더), `/diary/:date`, `/list`, `/chat`, `/stats`. (Option A에서는 hash 또는 단일 state로 대응.)
- 리스트→에디터→뒤로 = **리스트로** 복귀 (캘린더 아님).
- AI 채팅 인용 일기→에디터→뒤로 = **AI 채팅으로** 복귀.
- 리스트 복귀 시 마지막으로 보고 있던 달·정렬 상태 유지.
- AI 채팅 복귀 시 같은 대화 위치(스크롤)로 복귀.
- 모달은 history-stack에 들어가지 않는다 — 모달 닫힘이 화면 뒤로가기로 이어지지 않음.

## Non-Goals
- 딥링크·공유 URL — v2.
- 브라우저 새로고침 후 상태 완전 복구 — 직전 화면 식별 정도만 권장(가능 범위).

## Dependencies
REQ-001

## Suggested Scope
- Option B/C: React Router 채택 + scroll restoration 훅 + 화면별 미니 상태 영속화.
- Option A: 단순 state machine + 진입 경로(referrer) 메모.

## Expected Affected Areas
- Frontend: 라우팅 셸, scroll-restore 훅, 화면 상태 메모 헬퍼
- Backend: 없음
- Database: 없음
- Tests: history 스택 시뮬레이션 (리스트→에디터→뒤로 = 리스트)
- E2E: 권장 — 3개 진입 경로 각각의 뒤로가기 동작

## Risk Level
Medium — 뒤로가기 동작은 사용자가 즉시 체감하는 부분이라 버그 시 신뢰도 손실 큼.

## Suggested Test Coverage
- 3개 경로별 에디터 진입 → 뒤로 → 올바른 화면 + 보존 상태.
- 모달 열고 닫기가 history에 영향을 주지 않음.
